## Top Stories

### [Running Tesla Model 3's computer on my desk using parts from crashed cars](https://news.ycombinator.com/item?id=47523330)
A developer booted a Tesla Model 3 Media Control Unit and display on their desk using salvaged parts. The write-up details the struggle of reverse-engineering the wiring, noting that cars use massive bundled wiring looms rather than individual cables. The HN thread features automotive engineers discussing CAN bus hacking, with several pointing out that Tesla's bug bounty program grants permanent root access to researchers who find valid vulnerabilities.

### [My minute-by-minute response to the LiteLLM malware attack](https://news.ycombinator.com/item?id=47531967)
The engineer who discovered the recent supply chain attack on the `litellm` PyPI package shared their raw debugging transcript, showing how they used Claude to root-cause the malware. The malicious payload was hidden in a `.pth` file, which executes automatically on Python startup without requiring an explicit import. Commenters debated the dual nature of AI in security: it helps non-specialists detect threats faster, but also lowers the barrier for creating them.

### [Apple discontinues the Mac Pro](https://news.ycombinator.com/item?id=47535708)
Apple officially killed the Mac Pro, marking the end of its expandable workstation lineage. The transition to Apple Silicon—with its unified memory and lack of support for external PCIe GPUs—made the massive tower form factor practically obsolete, as the Mac Studio offers identical performance in a fraction of the size. The community mourned the final loss of user-serviceable, upgradable Macs.

### [$500 GPU outperforms Claude Sonnet on coding benchmarks](https://news.ycombinator.com/item?id=47533297)
A project called ATLAS demonstrates how a local model running on consumer hardware can beat frontier models on coding benchmarks using a multi-pass agentic harness. Instead of zero-shot generation, it generates multiple solutions, uses a smaller neural network to score their embeddings, and iteratively tests the most promising ones. HN users noted the catch: the verification process is incredibly slow (up to 20 minutes per task), making it impractical for interactive use.
