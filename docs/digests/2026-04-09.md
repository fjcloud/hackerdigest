## Top Stories

### [I ported Mac OS X to the Nintendo Wii](https://news.ycombinator.com/item?id=47691730)
A developer successfully ported Mac OS X to the Nintendo Wii, writing custom IOKit drivers (including a framebuffer) to run the OS on a PowerPC 750CL with just 88MB of RAM. It is a masterclass in reverse engineering and low-level systems programming that demonstrates the power of early OS-level abstractions. The community loved the detailed, AI-free writeup, especially the casual flex of the author debugging kernel panics from an economy class airplane seat.

### [LittleSnitch for Linux](https://news.ycombinator.com/item?id=47697870)
Objective Development released a free Linux version of their beloved macOS application firewall, Little Snitch, built on top of eBPF. It brings a polished, user-friendly outbound firewall to Linux, though it relies on eBPF which limits its ability to do deep packet inspection compared to the macOS version. Discussion centered on the trade-offs of eBPF's strict complexity limits, and the tension between the tool's closed-source daemon and the Linux community's preference for fully open alternatives like OpenSnitch.

### [Claude mixes up who said what](https://news.ycombinator.com/item?id=47701233)
Users are noticing a bug where Claude (particularly in the Claude Code CLI) confuses its own internal reasoning or background tool outputs with user input, leading it to hallucinate user commands. This highlights a fundamental vulnerability in agentic workflows where the lack of architectural boundaries between data and control paths can cause an LLM to confidently execute destructive actions it thinks the user authorized. Commenters debated whether this is a harness issue or a deeper model flaw caused by the lack of "speaker embeddings" in the token stream.

### [Open Source Security at Astral](https://news.ycombinator.com/item?id=47699181)
The team behind `uv` detailed their rigorous software supply chain security practices, including hash-pinning GitHub Actions and using Trusted Publishing. With recent supply chain attacks, it serves as a practical blueprint for securing open-source release pipelines. The thread highlighted the immense, inherent complexity of securing GitHub Actions, with some arguing that pinning hashes is merely security theater unless you own the underlying CI infrastructure.
