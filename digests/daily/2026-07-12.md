## Top Stories

### [Grok Build CLI uploads the entire repo by default, including .env, ignoring opt-out](https://news.ycombinator.com/item?id=48877371)
A reverse-engineered teardown of xAI's new coding CLI finds that even with "Improve the model" disabled, it exfiltrates every tracked file plus git history to a named GCS bucket, with secrets in `.env` transmitted unredacted. The teardown was largely LLM-written (j_bum and gruez flagged this) but the packet captures and bucket references check out. `gitgud` reads it as the natural endpoint of opaque native harnesses and recommends opencode + your-own-API; `gruez`'s counter -- that any cloud agent has the same capability via accumulated tool calls -- is the meatiest thread take. `thejazzman` adds the obvious implication: "so xAI now has a legal copy of all of Tesla's code," linking today's order from Musk telling Tesla staff to switch to Grok. The clearest practical response is `phaseleza`'s bubblewrap recipe: read-only `.git`, hidden sensitive dirs mounted empty, isolated netns with a Unix-socket HTTP proxy whitelisted to model hostnames.

### [Prefer strict tables in SQLite](https://news.ycombinator.com/item?id=48873940)
Evan Hahn's short case for `CREATE TABLE ... STRICT`. The thread is mostly long-time SQLite users relitigating the SQLite-vs-Postgres type-safety debate, but the actionable beats are real: `simonw` shipped `sqlite-utils transform --strict` during the thread (no `ALTER TABLE` path exists, you have to copy), `bch` shares a UUID-turned-octal bug story as a concrete lose, and `mort96` notes the missing piece -- there is still no proper timestamp/datetime type, just text the SDK is expected to keep ISO-and-UTC by convention. `Rendello` points out foreign keys are also off by default; a global `STRICT` pragma was considered and rejected.

### [We scaled PgBouncer to 4x throughput, no clients changed](https://news.ycombinator.com/item?id=48872874)
Clickhouse's managed-Postgres team runs four PgBouncer instances on the same VM bound to the same port via `SO_REUSEPORT` plus the new in-process "peering" for cancel-key routing. `__s` notes this dodges the userspace hop you'd get from `HAProxy` and the docs on the peering protocol (cancel tokens encoded with the originating bouncer's id) are linked in the thread. `saisrirampur` (the author) helpfully notes they considered `pgdog` and `yandex/odyssey` and chose PgBouncer for prepared-statement maturity + Postgres-nativeness; `JustSkyfall` runs pgdog in prod and finds the trade-off worth it for streaming workloads. `bradfitz`'s "I'm 46 and Postgres is still doing per-connection forks" is the canonical HN-veteran framing.

### [Show HN: Ant -- a JavaScript runtime and ecosystem from scratch](https://news.ycombinator.com/item?id=48875377)
A small C-based JS engine + npm-compatible package manager + their own registry, billed at "8MB including the runtime." The community reception is split: `pikuseru` and `EvilGoose` are excited about built-in sandboxing; `tekacs` digs up the March `cesanta/elk` issue showing the original codebase lifted from an AGPL project before the February rewrite (`theMackabu` acknowledges the dump and credits the cleanup). `colinhb`'s question is the right one -- what design bets unlock smaller-and-faster-vs-V8 -- and `CharlesW`'s answer ("looks like an LLM-rewritten Elk") is the through-line. Naming fights inevitable: Apache Ant, Ant Design, Anthropic's `ant` CLI all collided. Worth bookmarking on the v0 promise alone.

### [Mesh LLM: distributed AI inference over iroh](https://news.ycombinator.com/item?id=48876505)
iroh (the `number0`-derived QUIC dial-a-key networking lib) now hosts a mesh runner that splits big models across commodity boxes. Contributor `i386` (skippy engine author) posts real numbers in-thread: Qwen 235B A22B at ~16 tok/s across 2 nodes; GLM 5.2 at ~10 tok/s in his home lab over simulated 5ms WAN with jitter. Key open questions from the HN audience: `nullc` asks about MoE-aware expert routing to keep batching wins, `Creamsicle47` about privacy/payload visibility between mesh nodes (`tekacs` confirms iroh is QUIC-encrypted dial-a-key), `DerivativeBS` about mid-inference fault tolerance (no answer yet). Compared to llama.cpp's RPC layer, the framing is "suitable for builders with metro-latency LANs," not a global mesh.

## Projects & Tools

- **[RISCBoy](https://news.ycombinator.com/item?id=48876245)** -- Open-source portable games console (board + CPU + peripherals) by Raspberry Pi's Hazard3 author Luke Wren. GitHub's own blurb is a love letter: "Game Boy Advance from a parallel universe where RISC-V existed in 2001." Unique for being a fully-open ASIC-level design rather than another FPGA port.
- **[Biff.graph](https://news.ycombinator.com/item?id=48820361)** -- Lightweight subset of Pathom for treating your Clojure codebase as a queryable graph (resolvers keyed on output, automatic batch + caching). `geokon` is rewriting a production service with it and calls it "really a new coding paradigm." The `Pathom` terser-API wishlist (auto-registration, re-registration, better nested-query error viz) is the thread's lasting ask.
- **[Sqlsure](https://news.ycombinator.com/item?id=48875342)** -- Apache-2.0 semantic SQL linter that catches fan-out double-counting and similar additive-measure bugs *before* execution. Ran over BIRD + Spider gold answer keys, flagged 45 cases including a BIRD query the author proved wrong by 8x (filed upstream). MCP server mode for AI agents is included.
