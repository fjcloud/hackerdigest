## Top Stories

### [Anthropic: A global workspace in language models](https://news.ycombinator.com/item?id=48808002)
Anthropic publishes a mechanistic-interpretability paper identifying "J-Space" — an abstract reasoning subspace in Claude's middle layers where the model appears to represent future possible concepts before committing to output tokens. Their "J-Lens" technique reads this subspace, and they show counterfactual reflection training can shape internal thought without changing behavior. The HN thread is split: a serious contingent (Neel Nanda replicated on Qwen) treats it as a real interpretability advance, while critics call the anthropomorphizing tone "homeopathy" or "pseudo-scientific bullshit." The J-Lens tool itself is open-sourced, so it can be applied to open-weight models regardless of how you read the prose.

### [GLM 5.2 and the coming AI margin collapse](https://news.ycombinator.com/item?id=48809877)
Blog post arguing the price gap between frontier and open-weight models is about to collapse, using Z.ai's GLM 5.2 as the case study at ~$4.40/M output tokens vs Opus at 6-8x more. The author discloses Fireworks AI sponsored credit, which colors reception, but the technical observations hold up: Braintrust comparisons find GLM 5.2 competitive with Opus on long-context retrieval. Best HN pushback from `0xbadcafebee`: pricing per token matters less than tooling, platform features, and ecosystem lock-in — comparing GLM to Claude Code is like comparing Linux kernel to Windows. Separately, `throwdbaaway` points out cached input tokens are 90% of agentic API cost and can be made near-free with DeepSeek's MLA/CSA techniques, which the post ignores.

### [Januscape: Guest-to-Host Escape in KVM/x86 [CVE-2026-53359]](https://news.ycombinator.com/item?id=48807908)
First KVM guest-to-host escape working on both AMD and Intel, lurking in shadow-paging code from a 2010 commit until now. KVM maintainer `bonzini` confirms it's a variant of CVE-2026-46113 from last April — the new piece is a publicly weaponized PoC that can trigger a host kernel panic (full escape exploit is held back). Mitigation is the kernel module parameters `kvm_intel.nested=0` and `kvm_amd.nested=0`. The thread rehashes the usual debate over whether nested virtualization should be exposed to multi-tenant VMs at all.

### [OpenWrt One – Open Hardware Router](https://news.ycombinator.com/item?id=48808482)
The official reference platform for OpenWrt: MediaTek SoC, dual-band WiFi (no 6GHz), 1+2.5GbE, ~$84-$106. Comment consensus is that it's not the most powerful hardware, but it's the first router where OpenWrt "just works" out of the box with first-class images and a recent attended-sysupgrade button added to the UI. The thread doubles as a tour of the alternatives landscape: OpenWrt Two is in the works (WiFi 7, GL.iNet partnership), OPNSense on a cheap N100 box is the pro-tier step, and several commenters note plain Debian + nftables + hostapd gives the best learning curve for those willing to maintain it.

### [CoMaps – FOSS Offline Maps](https://news.ycombinator.com/item?id=48808928)
A year-old fork of Organic Maps triggered by an internal governance dispute: a small group of shareholders made financial decisions and partnership deals (Kayak integration) without contributor input. CoMaps positions itself as the more openly governed alternative. Practical user reviews are positive for navigation, but search is still noticeably worse than Google and live traffic is the missing feature keeping people on Google Maps. One unresolved HN thread question: the project's funding sources aren't fully transparent, and at least one commenter voiced concern about nation-state-backed open-source forks.

## Projects & Tools

- **[Linux on the Atari Jaguar](https://news.ycombinator.com/item?id=48808663)** -- Boot-to-Busybox Linux on the 68000-based Atari Jaguar with its original 2MB RAM and no specialized hardware. About 90 seconds to shell, on a recent upstream kernel — possible because m68k support is still in mainline Linux. ([github.com/cakehonolulu/linux_jag](https://github.com/cakehonolulu/linux_jag))
- **[OfficeCLI](https://news.ycombinator.com/item?id=48807225)** -- Open-source, single-binary CLI that lets AI agents read and edit Word, Excel, and PowerPoint files without installing Office. ECMA-376 compliance is the open question in-thread; a competitor project (SmallDocs, Elastic-licensed) drew attention to the licensing tradeoff.
- **[M/PC – A Concatenative OS](https://news.ycombinator.com/item?id=48809810)** -- Devine Lu Linvega's experimental OS that takes the Unix pipe model and pushes it toward a Forth-like stack discipline. Targets a 64KB-RAM board. Worth five minutes for the "what if the shell were a concatenative language?" thought experiment alone.

## Worth Reading

- **[NSA and IETF: Fairness](https://news.ycombinator.com/item?id=48811887)** -- DJB's latest in the ML-KEM TLS fight, alleging the IETF process is being steered toward documenting pure (non-hybrid) post-quantum key exchange. `tptacek` provides the sharpest counterweight: ML-KEM wasn't designed by NSA but by European academics (Schwabe et al.), and DJB is opposing documentation of the *possibility* of running pure ML-KEM in TLS — hybrid ECC+PQ is already the mainstream. The substantive cryptography is largely lost in process complaints from both sides.
- **[AMD Ryzen AI Halo – $4k AI Dev Kit](https://news.ycombinator.com/item?id=48805624)** -- The same Strix Halo hardware that sold for $1.5k-$2k last year, now repackaged at $4k because the LLM dev-kit market will bear it. LTT Labs benchmarks against a Mac Studio and DGX Spark: AMD loses on memory bandwidth (256 GB/s vs Mac's ~800 GB/s) but wins on flexibility (real x86, any OS, PCIe slot if you change cases). A `tptacek`-style "this isn't worth it" consensus crystallizes in the comments.
- **[OpenSSH 10.4 Released](https://news.ycombinator.com/item?id=48811373)** -- Notable addition: composite post-quantum signatures (ML-DSA 44 + Ed25519), not enabled by default; the underlying draft only just hit the IETF datatracker. PQ key agreement took ~3 years from OpenSSH adoption to default — don't expect signatures to be faster.
