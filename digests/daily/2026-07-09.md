## Top Stories

### [Decoding the obfuscated bash script on a Uniqlo t-shirt](https://news.ycombinator.com/item?id=48829312)
Akamai partnered with Uniqlo to print a real, runnable bash script on a "Peace for All" t-shirt. The script is a sine-wave text scroller with bilingual English/Japanese comments, designed by the artist to be deliberately hostile to OCR. The blog author ran it through multiple OCR pipelines (Tesseract, Claude, Android circle-to-search) and diffed the outputs; `raphlinus` corrected the post -- the font is Roboto Mono, not Consolas, and the typesetting is kerned. The designer confirms in a linked YouTube video the OCR-resistance was intentional. Cultural artifact of the week, and a nice stress test for vision models -- `underyx` reports Opus 4.8 reconstructed it in one shot.

### [TypeScript 7](https://news.ycombinator.com/item?id=48833715)
Microsoft ships TypeScript 7.0, the first release on the rewritten `tsgo` compiler (Go-based, not Rust). The headline numbers are eye-opening: vscode type-checks in 10.6s down from 125.7s (11.9x), sentry 8.9x, bluesky 8.7x. The Go-vs-Rust language choice keeps recurring in-thread (`hoppp` argues Go hits the sweet spot), and `m3h`'s speedup table is the cleanest summary. Tooling caveat: Vue/Astro/Svelte/Angular and ts-jest users will hit breakage in 7.0 -- the compiler API ships in 7.1.

### [Rewriting Bun in Rust](https://news.ycombinator.com/item?id=48837877)
Jarred Sumner's long-post on porting Bun's runtime from Zig to Rust using a pre-release of Anthropic's Fable model. Cost: $165k in tokens, 11 days start-to-green-tests, ~20% binary size reduction, ~5% perf gain, fixed several long-standing memory leaks. The trust subplot matters: `dfabulich` documents the bumpy rollout (13k instances of unsafe, no Miri coverage, exposed UB in safe Rust) and an HN comment from Sumner 9 days pre-merge saying "there's a very high chance all this code gets thrown out completely." The post doesn't apologise for the messaging; `pier25` won't touch Bun again until there's an LTS for the Zig branch.

### [John Deere owners will get the right to repair equipment under FTC settlement](https://news.ycombinator.com/item?id=48838876)
FTC settles with Deere for $1M collective to five states plus 10 years of compliance oversight -- Deere must provide diagnostic tools and parts to owners and independent shops. The HN split is sharp: `taurath` calls the fine "nothing" against ~$10B profit; `ggoo` notes the antitrust angle the original advocacy didn't surface; `trinsic2` links to fighttorepair's analysis arguing the settlement doesn't break the monopoly. Worth pairing with the September 2025 EU precedent and Louis Rossmann's Consumer Rights Wiki for the policy context.

### [GPT-Live lands the same day as Grok 4.5](https://news.ycombinator.com/item?id=48834405)
OpenAI ships GPT-Live-1: a native full-duplex audio model that delegates substantive questions to GPT-5.5 in the background. Simon Willison had preview access and confirms the voice mode is no longer pinned to a years-old model. The standout comment is the architectural one: `artdigital` asks why voice modes across Claude/Gemini/Grok/ChatGPT still don't allow tool use during speech; OpenAI confirms API access is imminent. Same day, xAI ships Grok 4.5 at $2/$6 (sub-200K context) -- trained on Cursor data and described in third-party testing as Opus 4.7-class, fast enough to compete on token efficiency.

## Projects & Tools

- **[Chatto goes open source](https://news.ycombinator.com/item?id=48833116)** -- Go backend (AGPLv3) plus Svelte frontend (Apache 2.0), self-hostable as a single binary with NATS + LiveKit alongside. The "spite-driven Slack alternative" market is crowded, but `simonw` flags the license choice as deliberate (AGPL backend stops SaaSS-cloning while Apache frontend lowers enterprise friction) and `moeffju` says it's noticeably snappier than Mattermost in daily use.
- **[Cloudflare Drop](https://news.ycombinator.com/item?id=48836233)** -- Drag-and-drop a folder, get a live URL for 60 minutes, claim it to keep it. Static-only, 100MB total / 25MB per file. The first HN comment is the right one: `Bender` asks how Cloudflare prevents the obvious abuse -- the answer is CSAM scanning on R2 plus takedown via the existing Workers pipeline. Netlify had this exact product a decade ago; `karlkloss` flags the perpetual, royalty-free licence grant in the ToS as the deal-breaker.
- **[Microsoft releases Flint, a visualization language for AI agents](https://news.ycombinator.com/item?id=48834924)** -- Higher-level declarative spec than Vega-Lite, compiles to ECharts. The design premise: AI agents reliably emit short, semantic-typed specs but get unreliable when forced to write ~5x verbose ECharts configs. Bundled MCP server so agents can invoke it directly. Reasonable critique in-thread: `theK` and `YuechenLi` argue stringly-typed JSON isn't optimal for agent output (vs a typed TS surface), but the snippet-length evidence holds up.
- **[cargo-nextest](https://news.ycombinator.com/item?id=48800596)** -- The author surfaces a refreshed landing page for the Rust test runner. Per-test process isolation by default (the "cattle, not pets" line that the author admits could be reworded), 3x speed on real projects, first-class CI integration. Worth knowing about if you've ever waited on `cargo test`.

## Worth Reading

- **[Unicode's transliteration rules are Turing-complete](https://news.ycombinator.com/item?id=48829797)** -- A short, clean demonstration that UTS #35 rewrite rules can express Collatz in three rules on stock ICU. `quotemstr` notes this is unsurprising if you remember Semi-Thue systems; `SonOfLilit` argues it's not the "send-a-Unicode-string-and-infinite-loop-the-host" variety since the spec's built-in rule sets terminate. Practical takeaway: run custom transliterators with hard limits like you'd run someone else's regex.
- **[Separating signal from noise in coding evaluations](https://news.ycombinator.com/item?id=48837396)** -- OpenAI's audit of SWE-Bench Pro finds ~30% of tasks are broken in ways that confound model rankings (underspecified prompts, verifier tests that encode a specific solution's implementation details). The community consensus is that DeepSWE and Cognition's Frontier-Code are now the trustworthy coding evals; `jjcm`'s "given $100 of API spend, how much can a model accomplish?" is the missing dashboard.
- **[Benchmarking coding agents on Databricks' multi-million line codebase](https://news.ycombinator.com/item?id=48837696)** -- Useful counterweight to synthetic benchmarks: a multi-million-line proprietary codebase across several languages. Findings worth quoting: harness and prompt tooling dominate over raw model choice; GLM 5.2 is the open-source standout; cheaper-per-token models (Sonnet 5) often consumed more total tokens because they took longer to converge. The blog's big recommendation -- "build your own benchmark" -- is the right one.
- **[Remote Attestation](https://news.ycombinator.com/item?id=48839397)** -- A readable intro to TPM-backed attestation and where it sits in the platform-security stack (under mTLS, SPIFFE, confidential computing). `zb3` and `michaelmrose` anchor the resistance thread, which is correct but tone-deaf to the corporate-infra target use case; `whiatp` adds a useful warning that physical-access attacks on a server's TPM-SPI bus remain a meaningful gap.
