## Top Stories

### [80386 microcode disassembled](https://news.ycombinator.com/item?id=48247004)
A reverse-engineering effort extracted and disassembled the microcode of the Intel 80386 processor from high-resolution die shots. The team trained a neural network to classify the presence of transistors to read the ROM bits. The HN discussion highlights the sheer complexity of the 386 compared to earlier chips, driven by features like virtual 8086 mode and memory paging.

### [.NET (OK, C#) finally gets union types](https://news.ycombinator.com/item?id=48234954)
C# is introducing discriminated unions in an upcoming preview, bringing a long-awaited functional programming feature to the language. While developers note the utility of modeling "this OR that" at the type level, some commenters point out that the initial implementation forces boxing for value types. The consensus is that getting the feature into the ecosystem is a net positive.

### [--dangerously-skip-reading-code](https://news.ycombinator.com/item?id=48246232)
A post exploring how AI coding tools might shift the unit of work from writing code to writing rigorous Markdown specifications. The author argues that as AI generates code faster than humans can read it, rigor must move to specs and tests. The comments are divided: some see this as the future of agentic engineering, while others view it as a doomed repeat of 1990s UML and CASE tool promises.

### [Microsoft open-sources "the earliest DOS source code discovered to date"](https://news.ycombinator.com/item?id=48253386)
Microsoft released the source code for DOS 1.00, which had to be OCR'd and transcribed from original paper printouts because no digital copies survived. The thread reflects on early PC history, with engineers noting the contrast to modern development where layers of virtualization obscure the underlying hardware.
