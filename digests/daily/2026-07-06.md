## Top Stories

### [Microsoft cuts 4,800 jobs, half from Xbox](https://news.ycombinator.com/item?id=48804401)
Microsoft's July transformation memo cuts 2.1% of headcount, with roughly half the reductions in Xbox as the unit "spins studios." The Verge and CNBC confirmed within minutes of each other; Microsoft's own blog post is the primary source. The mood on HN is grim resignation rather than surprise -- the new cognitive load is on execs blaming AI for the cuts, while veterans note these spreadsheets look identical to every prior round.

### [OpenPrinter](https://news.ycombinator.com/item?id=48797916)
Open Tools' crowdfunding campaign for a repairable, refillable inkjet printer using standard HP 63/302/803 cartridges crossed 1000 points on its second HN appearance, with a working prototype now printing color. The license is CC BY-NC-SA, which commenters flag as not actually open source by OSI definitions -- "Open as in OpenAI" became the recurring quip. The real pitch is repairability and no tracking dots: HP's Print Tracking Dots problem is the explicit motivation.

### [GPT-5.6 Sol Ultra coming to Codex](https://news.ycombinator.com/item?id=48799614)
OpenAI confirmed GPT-5.6 Sol Ultra will land in Codex, with an "ultra mode" that runs subagents cooperatively rather than just dispatching them serially. The thread got technical fast: one commenter dug into the Codex source and found "ultra" is largely a prompt-level alias for "max effort" plus a single line telling the model to delegate, unlike Claude Code's Dynamic Workflows which spin up an actual JS orchestrator. Several comment threads openly speculate this is a move to keep subscribers from churning to Anthropic once Fable access tightens.

### [Zuckerberg: AI agent development going slower than expected](https://news.ycombinator.com/item?id=48767058)
The Reuters piece surfaced a remarkably candid quote from Zuck admitting Meta's agent progress over the last four months "hasn't really accelerated in the way that we expected." The comment section turned into a postmortem on Meta's Llama falloff -- a top voted theory is that the team was cannibalized by management's pivot to "use our engineers to label data," which the engineers revolted against. A pointed comment from `efficax` matches the prevailing vibe: agents deliver 2-3x more code, but require 2-3x more review, so headcount goes sideways rather than down.

### [Cloudflare launches Workers Cache](https://news.ycombinator.com/item?id=48804014)
Cloudflare flipped the Workers architecture: now the cache can sit *in front of* the Worker, not just behind it. Configured with one block in `wrangler.jsonc` and standard `Cache-Control` headers, with first-class `stale-while-revalidate`, tag-based purge, and per-`ctx.props` cache keys for multi-tenant auth'd APIs. Several commenters said the marketing is overcooked but the technical design is genuinely novel -- cache stages can now sit *between* entrypoints inside a single Worker, letting you compose "auth outside, expensive work inside" without a separate service.

## Worth Reading

- **[Does code cleanliness affect coding agents? A controlled minimal-pair study](https://news.ycombinator.com/item?id=48798815)** -- SonarSource ran 660 trials through Claude Code. Surprisingly modest headline: cleanliness does *not* change pass rate, but it cuts token use 7-8% and reduces file revisits by 34%. Author `geraltofrivia` defends the methodology in-thread against accusations that "cleaning" via AI doesn't produce clean code (it's a deterministic rule-violation diff, not an LLM judgment call). Worth pairing with `minimaxir`'s refactor-prompts that improve baseline agent performance.
- **[The (Petty) Reason We Didn't End Up Using Jj](https://news.ycombinator.com/item?id=48804492)** -- Gradle's love letter to Jujutsu ending in a `gradlew.bat` line-ending blocker. `jj` doesn't read `.gitattributes`, so CRLF normalization on Windows produces a "six-figure file count of phantom changes." Gorgeous spec-by-spec postmortem of why one tiny gap kills a real adoption.
- **[Starring the Computer](https://news.ycombinator.com/item?id=48796093)** -- A visual history of personal computing with annotated computer families and machine photographs. The kind of resource that doesn't get built often and is hard to find elsewhere.
